Data Types

What Gets Remembered

Six categories of family data, each ingested and indexed differently but all searchable through a single natural-language interface.

Voice Conversations

Every spoken interaction is auto-transcribed via Whisper and saved as timestamped markdown. Speaker identification tags who said what. Full-text and semantic search across all conversations.

auto-transcribed indexed

Photos

Images are analyzed by the Vision LLM to generate descriptions, identify faces, detect objects, and tag events and dates. Original files stored locally with rich, searchable metadata.

face detection object tags event dates

Documents

Paper documents are scanned, OCR'd, and classified automatically. Medical records, receipts, contracts, school papers — all digitized, tagged by type, and full-text searchable.

scanned OCR'd classified

Calendar Events

Appointments, birthdays, deadlines, and reminders stored as structured data. Queryable by date range, person, or event type. Integrated with voice reminders and proactive alerts.

structured queryable

Notes

Freeform text captured via voice or typed input. Automatically linked to relevant people, topics, and time periods. Supports both quick capture and long-form journaling.

freeform linked

Decisions

Family decisions recorded with full context and rationale. Why did we choose this school? What contractor did we pick and why? Searchable decision history with reasoning chains preserved.

context rationale
Implementation

Phased Rollout

Family Memory is built incrementally. Each phase adds a new data modality while deepening the search and retrieval capabilities of previous phases.

Phase 1

Voice Conversation Logging & Search

The foundation. Every voice interaction is transcribed, timestamped, and indexed. Semantic search lets you ask natural-language questions about past conversations.

  • Whisper STT pipeline integration
  • Markdown conversation logs
  • ChromaDB vector indexing
  • Natural-language query interface
Phase 2

Photo Import with Vision LLM Tagging

Import photos from phones and cameras. The Vision LLM analyzes each image, generating descriptions, face tags, and event labels automatically.

  • Bulk photo import pipeline
  • Vision LLM analysis per image
  • Face recognition profiles
  • Event and date extraction
Phase 3

Document Scanning & Classification

Point the camera at a document or drop a PDF. OCR extracts text, the LLM classifies the document type, and everything is filed and indexed automatically.

  • Camera-based document capture
  • OCR text extraction
  • Auto-classification by type
  • Structured metadata extraction
Phase 4

Cross-Modal Search

The ultimate goal: search across all data types simultaneously. "Find the photo from when we discussed the renovation" connects voice logs to photo timestamps.

  • Unified search across all modalities
  • Temporal correlation engine
  • Cross-reference linking
  • Contextual result ranking
Tech Stack

What Powers Family Memory

All open-source, all running locally on the Jetson Orin Nano. Zero cloud dependency for any memory operation.

ChromaDB

Vector database for semantic search. Every piece of family data is embedded and stored for natural-language retrieval. The backbone of "just ask" queries.

vector search embeddings

Whisper STT

OpenAI's open-source speech-to-text model running locally. Converts voice conversations to text with high accuracy. Tiny and Small variants for speed vs. quality tradeoff.

voice transcription on-device

Vision LLM

Qwen2.5-VL or similar multimodal model for analyzing photos and documents. Generates descriptions, identifies objects and faces, reads text from images.

photo analysis document OCR

SQLite FTS5

Full-text search extension for structured queries. Handles date ranges, person filters, and keyword matching. Complements ChromaDB's semantic search with exact lookups.

full-text search structured queries

Markdown

The universal storage format. Every piece of data ultimately lives as a human-readable markdown file. Browse your family's memory with any text editor. No vendor lock-in, ever.

human-readable portable
Key Decisions

Design Philosophy

The architectural choices that define how Family Memory works and why it works that way.

"Markdown First"

All data is stored as human-readable markdown files, never locked in proprietary formats. If CUBE disappears tomorrow, your family's memory is still a folder of text files you can read with anything. Data portability is non-negotiable.

"Hybrid Search"

Vector search (semantic / ChromaDB) plus keyword search (BM25 / SQLite FTS5) combined for best recall. Semantic search finds conceptually related results; keyword search catches exact names, dates, and terms that embeddings might miss.

"Per-Person Profiles"

Each family member has a dedicated profile with their preferences, health information, goals, and history. The system tailors responses and proactive alerts per person. Profiles are editable markdown files — no hidden data.

"Year-Zero Problem"

The hardest part is starting. Family Memory has no value on day one. But the value compounds exponentially over time — a year of indexed conversations and photos creates a resource that cannot be recreated. Start collecting now; future you will thank present you.
