Email Integration
Full Gmail integration via the Google API stack. Read, classify, summarize, and send emails — all driven by voice commands or automated by the morning briefing.
GMAIL API — GOOGLE STACK
How It Works

Gmail API via google-api-python-client with OAuth2 authentication. Full programmatic access to your inbox without scraping or IMAP polling.

Read: Monitor inbox, classify emails by urgency using the LLM, and generate concise summaries for the morning briefing. Flag emails that need responses.

Send: Compose and send emails on voice command. “Email the contractor about the ramp measurements” → CUBE drafts, reads it back, sends on confirmation.

Dedicated address: Forward emails to cube@digitalsurfacelabs.com for automatic processing — invoices, receipts, notifications.

REAL-TIME MONITORING
IMAP Watch via Gmail Pub/Sub

Gmail Pub/Sub pushes notifications to CUBE in real time when new emails arrive. No polling, no wasted compute.

CUBE classifies incoming emails as they arrive:

  • → Urgent: Spoken alert immediately
  • → Action needed: Queued for next briefing
  • → Informational: Logged silently

All email summaries stored as markdown for future retrieval.

Tracking
Goals & Tracking
All goals stored as markdown files, reviewed automatically in the morning briefing, with proactive suggestions based on your patterns.

Health Goals

Workout frequency targets, nutrition goals, sleep tracking. CUBE correlates with Apple Health data and nudges you in the morning briefing. “You’ve worked out once this week — your goal is three. Try noon today, it’s 72°F and sunny.”

Project Goals

Active projects with tasks, deadlines, and dependencies. Each project is a markdown file with structured subtasks. CUBE surfaces the next action in the morning briefing and tracks completion over time.

Financial Goals

Budget tracking, bill reminders, and spending patterns. Forward receipts and invoices to CUBE for automatic categorization. Monthly summaries generated as markdown reports.

Habit Building

All goals feed into a proactive suggestion engine. CUBE learns your patterns over time and adjusts its briefings to be more relevant. Context-aware: weather, schedule, and energy level all factor in.

Decisions
Key Decisions
The architectural choices that shaped this module and the reasoning behind them.

Morning Briefing First

The morning briefing is the highest daily-utility feature and proves the CUBE concept immediately. If CUBE can wake you up with a useful spoken summary of your day, everything else follows. This is the “wow” moment.

Gmail API, Not IMAP

Gmail API provides better OAuth2 integration, richer metadata (labels, threads, importance markers), and push notifications via Pub/Sub. IMAP is a legacy protocol with polling overhead and limited classification data.

Voice-First Design

All daily usefulness interactions can be fully voice-driven with no screen required. Morning briefing is spoken. Tasks are created by voice. Email is dictated and confirmed verbally. The screen is optional, not primary.

Markdown Summaries

Every briefing is saved as a searchable markdown record. Ask CUBE “what did you tell me last Tuesday?” and it can retrieve the exact briefing. This creates a personal journal that grows over time — your life, indexed.
