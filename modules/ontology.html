Know Your Adversary
What CUBE Fights
Each institutional adversary profits from complexity. CUBE automates the work of navigating them: filing claims, following up, appealing denials, tracking deadlines, and extracting value that individuals leave on the table.
Health Insurance Payers $5K–$25K/yr
Deny first, delay second, hope you give up. Prior auths, EOB deciphering, surprise billing, out-of-network gotchas.
  • Auto-parse EOBs and match against provider bills
  • Draft and submit claim appeals from templates
  • Track prior authorization status and deadlines
  • Flag balance billing violations (No Surprises Act)
  • Compare in-network alternatives before procedures
  • Negotiate payment plans with provider billing
Tax System (IRS / State) $2K–$15K/yr
5,000+ page tax code that changes yearly. Miss a deduction and you overpay. Miss a deadline and you get penalized.
  • Ingest new tax code annually — extract rules automatically
  • OCR W-2s, 1099s, and supporting documents
  • Optimize deductions: itemized vs. standard, credits
  • Compute federal + state returns with audit trail
  • Track quarterly estimated payments for self-employed
  • Flag audit risk areas and suggest documentation
Medicare / Medicaid $3K–$20K/yr
Labyrinthine eligibility rules, Part A/B/C/D maze, enrollment windows, coverage gaps, and catastrophic coverage thresholds.
  • Determine eligibility across all program types
  • Compare Medicare Advantage vs. Original + Medigap
  • Track donut hole and catastrophic coverage thresholds
  • Flag enrollment windows and auto-remind
  • Appeal Part D formulary denials
  • Coordinate benefits between Medicare and employer plans
Drug Companies / Pharmacies $1K–$12K/yr
Opaque pricing, formulary games, step therapy requirements, manufacturer coupon mazes, and pharmacy benefit manager (PBM) middlemen.
  • Compare drug prices across pharmacies (GoodRx, CostPlus)
  • Find manufacturer patient assistance programs
  • Appeal step therapy and prior auth requirements
  • Track generic availability dates
  • Flag therapeutic alternatives with prescriber
  • Monitor formulary changes at open enrollment
Healthcare Providers $1K–$10K/yr
Chargemaster prices 10x Medicare rates, coding errors on 30–80% of bills, upcoding, and unbundling that nobody checks.
  • Audit medical bills against CPT/ICD coding rules
  • Compare charges against Medicare allowable amounts
  • Request itemized bills (triggers automatic reductions)
  • Negotiate cash-pay rates for uninsured procedures
  • Consolidate health records via FHIR from all providers
  • Track referrals, lab results, and follow-up appointments
Banks / Financial Services $500–$5K/yr
Overdraft fees, low savings rates, hidden account fees, and credit card rate games. They profit from your inattention.
  • Monitor account fees and flag avoidable charges
  • Track high-yield savings rates and recommend moves
  • Alert on credit card rate increases
  • Dispute unauthorized charges automatically
  • Optimize credit card rewards strategy
  • Track subscriptions and flag unused services
Airlines $200–$2K/yr
Dynamic pricing, change fees, lost bag runaround, delay compensation rules nobody knows, and loyalty program devaluations.
  • Monitor price drops after booking for rebooking/credit
  • File DOT complaints for delays and cancellations
  • Track EU261 compensation for international flights
  • Automate lost baggage claims with documentation
  • Compare loyalty program values across airlines
ISPs / Cell Providers $300–$2K/yr
Promotional rates that expire silently, data caps, equipment rental fees, and retention departments that only help when you threaten to leave.
  • Track promotional rate expirations
  • Generate plan comparison spreadsheets
  • Script retention department calls
  • File FCC complaints for service issues
  • Monitor usage against plan limits

Core Trade-off
Flexibility vs. Repeatability
A pure rule-based tax system breaks every April. A pure LLM system hallucinates deductions. CUBE uses a layered hybrid: the LLM reads new regulations and extracts structured rules, then a deterministic engine executes them. Humans review. Feedback improves extraction. The system gets better every year without being rewritten.

The Problem

Rules Rot

Tax code changes every year. Insurance formularies change every quarter. Step therapy requirements change monthly. Any hard-coded rule system requires constant manual maintenance by domain experts — which is the exact bottleneck we are trying to eliminate.

LLMs Hallucinate

An LLM asked “can I deduct my home office?” might say yes even when you don’t qualify. For high-stakes domains like taxes and healthcare, you need deterministic execution with an audit trail. “I did X because rule Y says Z” — not “the model thought so.”

What Gets Replaced

Patient names, SSNs, dates of birth, addresses, phone numbers, account numbers, employer names, hospital names, provider names, prescription details, diagnosis codes tied to identity. Everything that could identify a person.

What Stays the Same

Document structure, field types, value ranges, coding systems (CPT, ICD-10, tax form fields), edge cases (multi-state filing, secondary insurance, partial year coverage), and statistical distributions. The synthetic data triggers the same code paths.

Regulatory Landscape
Regulation & Access
How regulated each domain is determines CUBE’s approach. Heavily regulated domains (banking, healthcare) require careful API-based access. Lightly regulated domains (airlines, telecom) are more accessible but the rules change constantly. The phone system is the most universal access point — every institution has a phone number.
Banking / Financial
FDIC, SEC, CFPB, state regulators. Direct account access requires bank partnerships or screen scraping (Plaid). Read-only aggregation is feasible.
Approach: Plaid API for read access, manual entry fallback, no write access to accounts
Healthcare Data
HIPAA, HITECH, state privacy laws. But FHIR + patient right of access means individuals CAN get their data programmatically. 21st Century Cures Act requires it.
Approach: FHIR R4 APIs (patient-authorized), SMART on FHIR auth, OCR for paper documents
Tax / Government
IRS e-file requires authorized provider status. State varies. But document processing and computation are fully local — no API needed until final submission.
Approach: OCR + local computation, e-file via authorized transmitter or print-and-mail
Insurance (Health)
State DOI, ACA, ERISA (employer plans). Claims data available via payer portals. No standardized API, but EOBs and letters can be OCR’d.
Approach: OCR all documents, phone calls for follow-up, appeal letters via templates
Telecom / ISP
FCC, minimal enforcement. Plans and pricing are public. Main leverage is threatening to switch and filing FCC complaints. Phone calls are the primary tool.
Approach: Plan scraping, retention call scripts, FCC complaint filing, price monitoring
Airlines / Travel
DOT, EU261. Pricing is public. Booking APIs exist (Google Flights, Amadeus). Compensation claims are well-defined but rarely filed.
Approach: Price monitoring APIs, DOT complaint forms, EU261 claim templates

The Phone Is the Universal API

Every institution — no matter how regulated — has a phone number. CUBE can place calls via Twilio, navigate IVR menus, wait on hold, and escalate to humans. For domains where APIs don’t exist or are locked behind partnerships, the phone is CUBE’s universal access layer. Less regulated than bank APIs, more reliable than web scraping, and available for every single adversary on this page.

Foundation
The Markdown Computer
CUBE is a new type of computer where Markdown is the operating system. Every piece of state — user profiles, decisions, rules, feedback, health records, tax computations — is a human-readable Markdown file. The system explains itself because it IS the explanation.
HUMAN-READABLE STATE

No Black Boxes

Every decision, computation, and recommendation is stored as a Markdown file that any human can read, understand, and correct. There are no opaque databases or hidden model weights driving behavior.

  • Tax computations with cited IRC sections
  • Medical bill audits with CPT code references
  • Insurance appeal letters with regulation citations
  • User corrections stored inline with context
CONVERSATIONAL INTERFACE

Talk to Your Data

The primary interface is conversation — voice or text. Ask questions, give corrections, request actions. CUBE searches its Markdown knowledge base, reasons about it, and responds with citations.

  • “Why did you deduct $4,680 for home office?”
  • “Appeal that denied claim from Aetna.”
  • “What’s my total healthcare spend this year?”
  • “Remind me when open enrollment starts.”
MULTI-USER

Family-Aware

CUBE knows who is talking to it. Each family member has their own profile with preferences, context, and correction history. Shared context (household taxes, family health) is accessible to authorized members.

  • Voice identification or explicit “this is Joe”
  • Per-person communication preferences
  • Shared household financial context
  • Individual health records (private by default)
SELF-IMPROVING

Gets Better Over Time

Every interaction is a training signal. Corrections improve future behavior. Successful outcomes reinforce patterns. The system’s quality is proportional to how much you use it.

  • Correction log per domain per person
  • Outcome tracking (did the appeal work?)
  • Rule accuracy metrics over time
  • Proactive suggestions based on past patterns
