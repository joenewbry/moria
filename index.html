04 ‚Äî Software Architecture
Open Source All The Way Down
Everything runs locally. Zero cloud dependency. Zero subscription cost. Just electricity. The entire stack uses markdown as the thinking layer so interactions are human-readable and inspectable.
L0 ‚Äî OS & Runtime

JetPack 6.x + Ubuntu 22.04

NVIDIA's optimized Linux distribution with CUDA, cuDNN, and TensorRT pre-installed. Container runtime via Docker for isolation and easy updates.

JetPack 6.2 Ubuntu 22.04 Docker CUDA 12.x
L1 ‚Äî Inference Engine

Ollama + llama.cpp (TensorRT-LLM optional)

Ollama provides the model serving layer ‚Äî easy model management, hot-swapping, and an OpenAI-compatible API. Under the hood, llama.cpp handles 4-bit quantized inference optimized for the Orin's Ampere GPU.

Ollama llama.cpp TensorRT-LLM GGUF Q4
L2 ‚Äî Models

Multi-Model Router

Fast router selects the right model per task. A 3B reasoning model handles conversation, a 3B vision-language model processes camera input, and a tiny 0.5B model runs always-on wake-word detection. All open-weight, all free.

Llama 3.2 3B Qwen2.5-VL 3B Phi-3 Mini Whisper Tiny (STT) Piper (TTS)
L3 ‚Äî Memory & Knowledge

ChromaDB + Markdown Knowledge Graph

All interactions, observations, and decisions are stored as markdown files ‚Äî human-readable, inspectable, and searchable. ChromaDB provides vector embeddings for semantic search. This is how CUBE "remembers" and learns.

ChromaDB Markdown Files SQLite (metadata) SentenceTransformers
L4 ‚Äî Voice Pipeline

Whisper ‚Üí LLM ‚Üí Piper TTS

Continuous audio loop: Whisper (tiny/small) transcribes speech locally, the LLM processes intent and generates a response, and Piper converts text back to natural speech. Runs entirely on-device with sub-second latency for short queries.

Whisper Piper TTS VAD (Silero) Wake Word
L5 ‚Äî Agent Framework

Task Engine + Tool Use

Inspired by human executive function: the agent maintains a task queue, breaks goals into steps, uses tools (email, calendar, web, phone), and logs every decision in markdown with reasoning chains. Users can ask "why did you do this?" and get a readable answer.

Task Queue Tool Registry Reasoning Log MCP Protocol
L6 ‚Äî Interface

Open WebUI + Voice + Camera

Three interaction modes: conversational voice (always listening for wake word), web dashboard accessible from any device on the local network, and camera-based visual context. All state is shared across interfaces.

Open WebUI Voice Loop Camera Feed REST API

Interaction States

IDLE

Waiting

Wake word armed, minimal compute. Silero VAD runs on the audio loop at low power. No LLM inference active.

LISTENING

Capturing Speech

VAD detected voice activity. Whisper STT is streaming the audio buffer. LED ring pulses green to signal active listening.

PROCESSING

Thinking

Whisper transcript complete. Intent router classifies the primitive type. LLM runs with RAG context injected from ChromaDB.

RESPONDING

Speaking

Piper TTS converts LLM output to audio. Streamed to speaker in real time. LED pulses in sync with speech amplitude.

CONFIRMING

Awaiting Reply

CUBE has proposed an action and is waiting for yes/no or clarification. Task is held in pending state. Times out after 30s.

ACTING

Background Task

A confirmed task is executing asynchronously (call, email, etc.). CUBE resumes IDLE. Notifies on completion via speaker alert.

Context Persistence

üíæ

Voice State ‚Üí Memory

Every RESPONDING and ACTING outcome writes to a markdown session log. The transcript, intent classification, and tool calls are preserved verbatim for future RAG retrieval.

PERSISTENT
üìã

Task Queue Bridge

COMMAND primitives spawn structured task objects in the task queue with all parameters captured from the voice context. The queue is persisted to disk ‚Äî CUBE remembers pending tasks across reboots.

DURABLE
üîç

RAG Injection

Before every LLM call, the intent router queries ChromaDB with the transcript text. Relevant memory chunks ‚Äî health records, past conversations, user preferences ‚Äî are injected into context automatically.

CONTEXTUAL
üåô

Nightly Compression

Each night, voice session logs are summarized into compressed markdown knowledge files. Raw transcripts are archived. The compressed summaries become the primary RAG source for future sessions.

VISION

Key Design Principles

Privacy by Architecture

Data never leaves the device. No cloud accounts, no telemetry, no subscriptions. Your data is literally inside a box on your shelf.

Transparent Reasoning

Markdown-first thinking means every decision is readable. No black-box magic ‚Äî ask why, get a real answer.

Zero Marginal Cost

All open-source models, all local compute. The only ongoing cost is electricity (~15W = ~$2/month). No subscriptions, ever.

Feels From The Future

The aluminum cube with the glowing lens should feel like a sci-fi prop that actually works. Industrial design matters for mass adoption.
